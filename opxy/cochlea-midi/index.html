2 · the drift — why no tuning table can hold this

Every chord is pure just intonation, but each lap the home note is carried a whole comma off where it began — so the reference pitch drifts without ever returning. A tuning table is set once and can't follow it; only a bend recomputed against the moving tonic on every note can. Bends are recomputed continuously below.

5 · MIDI stream

on this fork

Path B, hardest case. TRITAVA proves a scale a tuning table can't hold (no octave). COCHLEA proves a tuning that a table can't track: it moves. Stack pure fifths and thirds around Euler's lattice and the arithmetic refuses to close — a I–vi–ii–V–I of perfectly tuned chords returns a syntonic comma (81/80, ≈21.5¢) below where it started, and sinks another comma every lap. This drift is the reason keyboards are tempered at all.

COCHLEA refuses to temper. The model here is copied verbatim from the machine — the same lattice walk, the same named pumps, the same comma arithmetic over {2,3,5,7}. Only the output changed: for every chord tone the fork recomputes the exact just frequency against the current, drifted tonic, finds the nearest key, and bends to it. Watch the bend values sweep as the comma accumulates — that sweep is the drift, and it is exactly what a static tuning table cannot reproduce. If the OP–XY ever exposes real-time MTS SysEx retuning, that would be the ideal transport for this; per-note bend is the path that works today.
